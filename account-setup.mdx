Before you launch your first GPU instance, take a few minutes to complete your GPU Cloud account setup. A properly configured account ensures uninterrupted access to compute resources, gives you programmatic control through the API, and lets you collaborate safely with teammates through role-based permissions. This guide walks you through every step: registering and verifying your account, adding a payment method and billing alerts, generating and securing API keys, and inviting team members with the appropriate roles.

---

## Account Registration

<Steps>
  <Step title="Sign Up">
    Go to [console.gpucloud.com](https://console.gpucloud.com) and click **Sign Up**. Fill in your name, email address, and a strong password, then click **Create Account**.
  </Step>
  <Step title="Verify Your Email">
    Check your inbox for an email from GPU Cloud with the subject **Verify your email address**. Click the verification link inside. The link expires after 24 hours — if it expires, request a new one from the login page.
  </Step>
  <Step title="Complete Your Profile">
    After verification, log in and navigate to **Settings → Profile**. Add your organization name and time zone. These settings personalize your Console experience and appear on invoices.
  </Step>
</Steps>

## Billing & Payment

GPU Cloud uses pay-as-you-go pricing by default, so you need a valid payment method on file before launching instances. Reserved instance pricing is also available for predictable, long-running workloads.

### Add a Credit Card

<Steps>
  <Step title="Open Billing Settings">
    In the Console, go to **Settings → Billing**.
  </Step>
  <Step title="Add a Payment Method">
    Click **Add Payment Method**. Enter your credit or debit card details in the secure payment form. GPU Cloud uses Stripe for payment processing — your full card number is never stored on GPU Cloud servers.
  </Step>
  <Step title="Set a Default Payment Method">
    If you add multiple cards, click **Set as Default** next to the card you want charged automatically each billing cycle.
  </Step>
</Steps>

### Billing Alerts

<Tip>
  Set a monthly spending alert as soon as you add your payment method. Navigate to **Settings → Billing → Alerts**, click **Create Alert**, and enter a dollar threshold. GPU Cloud will email you when your accrued spend reaches that amount — a simple safeguard against runaway costs from forgotten instances.
</Tip>

GPU Cloud charges your card at the end of each calendar month. You can download itemized invoices at any time from **Settings → Billing → Invoices**.

## API Keys

The GPU Cloud REST API lets you automate instance management, storage provisioning, and team administration. Every API request must include a valid API key in the `Authorization` header.

### Generate an API Key

<Steps>
  <Step title="Open API Keys Settings">
    Navigate to **Settings → API Keys** in the Console.
  </Step>
  <Step title="Create a New Key">
    Click **Generate API Key**. Enter a descriptive name (for example, `ci-pipeline-prod` or `local-dev`) so you can identify the key's purpose later.
  </Step>
  <Step title="Choose Scopes">
    Select the permission scopes your key needs:

    | Scope | Access Granted |
    |---|---|
    | `instances:read` | List and view instance details |
    | `instances:write` | Launch, stop, and terminate instances |
    | `storage:read` | List volumes and snapshots |
    | `storage:write` | Create, attach, and delete volumes |
    | `team:read` | View team members and roles |
    | `team:write` | Invite and remove team members |

    Follow the principle of least privilege — grant only the scopes your integration actually needs.
  </Step>
  <Step title="Copy and Store Your Key">
    Click **Create**. Your key is displayed **once** — copy it immediately and store it in a secrets manager, password vault, or environment variable. GPU Cloud cannot retrieve the key after you close this dialog.
  </Step>
</Steps>

<Warning>
  Never commit API keys to source control or embed them in client-side code. If a key is exposed, revoke it immediately from **Settings → API Keys** by clicking **Revoke** next to the compromised key, then generate a replacement. Treat API keys with the same care as passwords.
</Warning>

### Use Your API Key

Pass the key as a Bearer token in the `Authorization` header of every API request:

<CodeGroup>
```bash cURL
curl https://api.gpucloud.com/v1/instances \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```python Python
import requests

headers = {"Authorization": "Bearer YOUR_API_KEY"}
response = requests.get("https://api.gpucloud.com/v1/instances", headers=headers)
print(response.json())
```

```typescript TypeScript
const response = await fetch("https://api.gpucloud.com/v1/instances", {
  headers: { Authorization: "Bearer YOUR_API_KEY" },
});
const data = await response.json();
console.log(data);
```
</CodeGroup>

## Team Members

GPU Cloud supports shared team workspaces so multiple users can access the same instances, storage volumes, and billing account under one organization.

### Invite a Team Member

<Steps>
  <Step title="Open Team Settings">
    Navigate to **Settings → Team**.
  </Step>
  <Step title="Send an Invitation">
    Click **Invite Member**, enter the person's email address, and select a role. Click **Send Invite**. They will receive an email with a link to join your team.
  </Step>
  <Step title="Manage Pending Invites">
    Pending invitations appear in a separate list on the Team page. You can resend or revoke any invite before it is accepted.
  </Step>
</Steps>

### Roles & Permissions

GPU Cloud provides three built-in roles. Assign the most restrictive role that still allows the team member to do their job.

| Role | Description |
|---|---|
| **Owner** | Full administrative access. Can manage billing, delete the organization, and transfer ownership. Only one Owner per organization. |
| **Admin** | Can invite and remove Members, manage API keys, launch and terminate instances, and configure storage. Cannot access billing settings or transfer ownership. |
| **Member** | Can view and use existing instances and storage volumes assigned to them. Cannot launch new instances, manage team members, or access billing. |

<Note>
  You can change a team member's role at any time from **Settings → Team** by clicking the role dropdown next to their name. Role changes take effect immediately.
</Note>
