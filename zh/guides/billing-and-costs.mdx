GPU Cloud 采用透明的按用量付费模式，没有前期承诺、没有隐藏费用。算力从实例进入 **Running** 状态起按秒计费，直到你停止或终止实例——你只为实际使用的部分付费。本指南解释账单上会出现的每个项目，教你在启动前估算成本，并带你了解在控制台中监控支出、避免账单意外的工具。

## 计费方式

### 算力——按秒计费

实例的算力费用会按 GPU 类型和实例中 GPU 数量按秒累计。计费从实例状态变为 **Running** 时开始，在你 **Stop** 或 **Terminate** 时立即停止。

<Note>
  **Stopped 与 Terminated 的区别：** *stopped* 状态的实例暂停算力计费但保留根盘（你会继续为存储付费）。*terminated* 状态的实例会永久删除根盘并停止所有费用。挂载的持久卷在你终止实例时**不会**被删除——它们仍然可用，并会继续产生存储费用，直到你显式删除它们为止。
</Note>

### 存储——按 GB / 月

存储按预配置的卷大小计费，而非按实际写入的数据量。计费精确到秒。

| 存储类型 | 价格 |
|---|---|
| 根盘（NVMe SSD） | 按 GB/月，按比例计算 |
| 持久 NVMe 卷 | 按 GB/月，按比例计算 |
| 持久 network-attached 卷 | 按 GB/月，按比例计算（价格更低） |
| 磁盘镜像快照 | 按 GB/月（压缩后大小） |

### 网络出站流量

- **入站（ingress）：** 始终免费
- **同区域出站：** 免费（例如同区域内实例 ↔ 卷）
- **出站到互联网：** 超出各账户等级每月免费额度的部分按 GB 计费

<Tip>
  将存储卷保持在与实例相同的区域，可避免跨区域传输费用并获得最低延迟。
</Tip>

## 成本估算示例

假设你运行一台 A100 80 GB 实例进行 72 小时的训练任务，并挂载了 1 TB 的 NVMe 持久卷。

| 项目 | 计算方式 | 费用 |
|---|---|---|
| A100 80GB 算力 | 72 小时 × $2.21/hr | $159.12 |
| 根盘（100 GB NVMe） | 100 GB × 3 天/30 天 × $0.12/GB-mo | $1.20 |
| 持久卷（1 TB NVMe） | 1,000 GB × 3 天/30 天 × $0.10/GB-mo | $10.00 |
| 出站流量（< 免费额度） | — | $0.00 |
| **总计** | | **$170.32** |

<Note>
  以上价格仅为示意。实际价格请参见 [GPU Cloud 定价页面](https://gpucloud.com/pricing)，并在启动向导中实时显示。价格可能因区域而异。
</Note>

## 预留实例

对于可预测的长期工作负载，预留实例可提供高达 60% 的折扣（相较按需价格），但需承诺 1 个月或 3 个月的使用期。预留实例按月计费，无论实例是否处于运行状态。可从控制台的 **Billing → Reservations** 页面查看可用预留并购买。

## 查看当前支出

<Steps>
  <Step title="打开计费面板">
    在控制台侧边栏点击 **Billing**。**Overview** 标签页会显示当前计费周期的支出，并按资源类别（算力、存储、出站流量）细分。
  </Step>

  <Step title="按资源查看详细费用">
    点击 **Usage** 查看每个实例、卷和快照的逐行明细及其单独累计费用。使用日期范围选择器查看历史周期。
  </Step>

  <Step title="下载费用报告">
    在 Usage 页面点击 **Export** 下载 CSV 使用数据。每行包含资源 ID、资源名称、区域、起止时间、数量、单价和总费用。
  </Step>
</Steps>

## 设置支出提醒

支出提醒会在当前计费周期内的累计费用超过你设置的阈值时，发送邮件（以及可选的 webhook）。

<Steps>
  <Step title="进入支出提醒">
    在控制台中，前往 **Billing → Alerts**。点击 **Create Alert**。
  </Step>

  <Step title="设置阈值">
    输入一个金额（例如 `$500`）。你可以在不同阈值上创建多个提醒——例如在 $250、$500 和 $1,000 各设一个。
  </Step>

  <Step title="选择通知渠道">
    选择一个或多个渠道：
    - **Email** ——通知所有账户所有者和计费管理员
    - **Webhook** ——向你指定的 URL 发送 JSON POST（适用于 Slack、PagerDuty 等）
  </Step>

  <Step title="保存提醒">
    点击 **Save**。提醒立即生效。当阈值被跨越时，你会在几分钟内收到通知，而不是到计费周期结束时才收到。
  </Step>
</Steps>

<Warning>
  支出提醒仅是通知——它**不会**在阈值被跨越时自动停止你的实例。如果你需要强制预算控制，请使用 **Billing → Settings** 下的 **Spending Limit** 功能，达到限额后会暂停新的实例启动（但不会停止现有正在运行的实例）。
</Warning>

## 查看账单

GPU Cloud 会在每个自然月结束时生成一份账单。你可以在 **Billing → Invoices** 中查看所有账单。每份账单包含：

- **账单编号与计费周期**（例如 October 1–31, 2024）
- **账户与付款信息**
- 按资源类型（算力、存储、网络）分组的**明细**
- **已应用抵扣**（促销抵扣、推荐抵扣、支持抵扣）
- **小计、税费（如适用）与应付总额**

账单以 PDF 形式提供，可用于会计和报销。

## 付款方式

GPU Cloud 支持以下付款方式：

<CardGroup cols={2}>
  <Card title="信用卡 / 借记卡" icon="credit-card">

## 抵扣与促销

GPU Cloud 抵扣会在扣款前应用到你的账户余额。抵扣不可转让，并在 **Billing → Credits** 中显示的日期到期。

- **试用抵扣** ——新账户会获得一次性抵扣以体验平台
- **推荐抵扣** ——被推荐用户消费达到 $100 时，你可获得抵扣
- **支持抵扣** ——支持团队因服务中断或计费错误发放的抵扣
- **促销抵扣** ——在 **Billing → Redeem Code** 中兑换促销码时自动应用

## 常见问题

**如果我的付款失败会怎样？**
GPU Cloud 会在 24 小时后重试扣款，72 小时后再重试一次。如果所有重试均失败，正在运行的实例会被停止（而非终止），你会收到邮件通知。请在 **Billing → Payment Methods** 更新付款方式以恢复服务。

**已停止的实例会产生费用吗？**
实例停止后算力计费停止。你会继续为根盘存储付费，直到你终止实例。

**能否为误扣的费用申请退款？**
请在扣款后 7 天内联系 [support@gpucloud.com](mailto:support@gpucloud.com)。是否可退款将按具体情况评估。

**如何删除不再使用的卷？**
前往 **Storage → Volumes**，选择该卷并点击 **Delete**。卷必须处于 **Available**（未挂载）状态。删除立即生效且不可恢复。
