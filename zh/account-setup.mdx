在启动你的第一个 GPU 实例之前，请花几分钟完成 GPU Cloud 账户设置。正确配置的账户可以确保你不间断地访问计算资源，通过 API 获得程序化控制能力，并借助基于角色的权限与团队成员安全协作。本指南将引导你完成每一步：注册并验证账户、添加付款方式与计费提醒、生成并保护 API 密钥，以及以合适的角色邀请团队成员。

---

## 账户注册

<Steps>
  <Step title="注册">
    前往 [console.gpucloud.com](https://console.gpucloud.com) 并点击 **Sign Up**。填写你的姓名、邮箱地址以及一个强密码，然后点击 **Create Account**。
  </Step>
  <Step title="验证邮箱">
    在收件箱中查找主题为 **Verify your email address** 的 GPU Cloud 邮件，点击其中的验证链接。链接会在 24 小时后失效 —— 如果过期，可以在登录页面重新申请。
  </Step>
  <Step title="完善你的个人资料">
    验证完成后，登录并前往 **Settings → Profile**。填写你的组织名称和时区。这些设置会个性化你的控制台体验，并显示在账单上。
  </Step>
</Steps>

## 计费与付款

GPU Cloud 默认采用按用量付费模式，因此在启动实例之前你需要绑定一个有效的付款方式。对于可预期的长期运行工作负载，还提供预留实例定价。

### 添加信用卡

<Steps>
  <Step title="打开计费设置">
    在控制台中，前往 **Settings → Billing**。
  </Step>
  <Step title="添加付款方式">
    点击 **Add Payment Method**。在安全的付款表单中输入你的信用卡或借记卡信息。GPU Cloud 使用 Stripe 处理付款 —— 你的完整卡号绝不会存储在 GPU Cloud 的服务器上。
  </Step>
  <Step title="设置默认付款方式">
    如果你添加了多张卡，可以点击你希望在每个计费周期自动扣款的那张卡旁边的 **Set as Default**。
  </Step>
</Steps>

### 计费提醒

<Tip>
  在添加付款方式后请立即设置每月支出提醒。前往 **Settings → Billing → Alerts**，点击 **Create Alert**，输入一个金额阈值。当你的累计消费达到该金额时，GPU Cloud 会通过邮件通知你 —— 这是一个防止因遗忘的实例导致成本失控的简单保障。
</Tip>

GPU Cloud 会在每个自然月末从你的卡上扣款。你可以随时从 **Settings → Billing → Invoices** 下载详细账单。

## API 密钥

GPU Cloud REST API 让你可以自动化实例管理、存储配置和团队管理。每次 API 请求都必须在 `Authorization` 请求头中包含一个有效的 API 密钥。

### 生成 API 密钥

<Steps>
  <Step title="打开 API 密钥设置">
    在控制台中前往 **Settings → API Keys**。
  </Step>
  <Step title="创建新密钥">
    点击 **Generate API Key**。输入一个描述性名称（例如 `ci-pipeline-prod` 或 `local-dev`），方便你日后识别密钥用途。
  </Step>
  <Step title="选择权限范围">
    选择密钥所需的权限范围：

    | 范围 | 授予的访问权限 |
    |---|---|
    | `instances:read` | 列出并查看实例详情 |
    | `instances:write` | 启动、停止和终止实例 |
    | `storage:read` | 列出卷和快照 |
    | `storage:write` | 创建、挂载和删除卷 |
    | `team:read` | 查看团队成员和角色 |
    | `team:write` | 邀请和移除团队成员 |

    请遵循最小权限原则 —— 只授予集成实际需要的权限范围。
  </Step>
  <Step title="复制并妥善保存你的密钥">
    点击 **Create**。密钥**仅显示一次** —— 请立即复制并存入密钥管理器、密码保险箱或环境变量中。关闭该对话框后，GPU Cloud 将无法再检索该密钥。
  </Step>
</Steps>

<Warning>
  切勿将 API 密钥提交到源代码版本控制系统或嵌入到客户端代码中。如果密钥泄露，请立即在 **Settings → API Keys** 中点击对应密钥旁的 **Revoke** 撤销它，然后生成新的密钥。请像对待密码一样谨慎对待 API 密钥。
</Warning>

### 使用你的 API 密钥

在每次 API 请求的 `Authorization` 请求头中，以 Bearer 令牌形式传递密钥：

<CodeGroup>
```bash cURL
curl https://api.gpucloud.com/v1/instances \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```python Python
import requests

headers = {"Authorization": "Bearer YOUR_API_KEY"}
response = requests.get("https://api.gpucloud.com/v1/instances", headers=headers)
print(response.json())
```

```typescript TypeScript
const response = await fetch("https://api.gpucloud.com/v1/instances", {
  headers: { Authorization: "Bearer YOUR_API_KEY" },
});
const data = await response.json();
console.log(data);
```
</CodeGroup>

## 团队成员

GPU Cloud 支持共享的团队工作空间，让多位用户可以在同一个组织下访问相同的实例、存储卷和计费账户。

### 邀请团队成员

<Steps>
  <Step title="打开团队设置">
    前往 **Settings → Team**。
  </Step>
  <Step title="发送邀请">
    点击 **Invite Member**，输入对方的邮箱地址，并选择一个角色。点击 **Send Invite**。对方将收到一封邮件，其中包含加入你团队的链接。
  </Step>
  <Step title="管理待处理邀请">
    待处理的邀请会显示在团队页面的一个单独列表中。你可以在邀请被接受之前重新发送或撤销它。
  </Step>
</Steps>

### 角色与权限

GPU Cloud 提供三种内置角色。请为每位团队成员分配在其能完成工作前提下最受限的角色。

| 角色 | 描述 |
|---|---|
| **Owner** | 完全的管理权限。可以管理计费、删除组织和转移所有权。每个组织只能有一位 Owner。 |
| **Admin** | 可以邀请和移除 Member、管理 API 密钥、启动和终止实例并配置存储。无法访问计费设置或转移所有权。 |
| **Member** | 可以查看和使用已分配给他们的现有实例和存储卷。无法启动新实例、管理团队成员或访问计费。 |

<Note>
  你可以随时在 **Settings → Team** 中点击团队成员姓名旁的角色下拉框来更改其角色。角色变更立即生效。
</Note>
