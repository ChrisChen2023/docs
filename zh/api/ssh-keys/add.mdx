---
title: "通过 API 向 GPU Cloud 账户添加 SSH 密钥"
sidebarTitle: "添加 SSH 密钥"
description: "向你的 GPU Cloud 账户注册一个新的 SSH 公钥，以便在启动实例时启用 SSH 访问。支持 RSA 和 Ed25519 密钥类型。"
---

在你通过 SSH 连接到 GPU Cloud 实例之前，需要向账户注册至少一个公钥。此端点接收密钥完整的公钥内容以及一个易于识别的名称 —— 因此你可以区分来自不同机器或团队成员的密钥。添加完成后，该密钥即可附加到你启动的任何新实例上。

## 端点

```
POST https://api.gpucloud.com/v1/ssh-keys
```

## 请求体

<ParamField body="name" type="string" required>
  用于描述此密钥的标签（例如 `"my-macbook"` 或 `"ci-pipeline"`）。此名称会显示在仪表板和 API 响应中，帮助你识别密钥。
</ParamField>

<ParamField body="public_key" type="string" required>
  你的 SSH 公钥的完整内容，与 `.pub` 文件中显示的完全一致（例如 `"ssh-ed25519 AAAAC3Nza... user@host"`）。支持 RSA 和 Ed25519 密钥类型。
</ParamField>

## 响应字段

<ResponseField name="id" type="string">
  分配给新注册 SSH 密钥的唯一标识符。
</ResponseField>

<ResponseField name="name" type="string">
  你为此密钥提供的标签。
</ResponseField>

<ResponseField name="fingerprint" type="string">
  从你的公钥派生出的 SHA-256 指纹。可使用它验证是否上传了正确的密钥。
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 时间戳，表示该密钥被注册到你账户的时间。
</ResponseField>

## 请求示例

```bash
curl -X POST https://api.gpucloud.com/v1/ssh-keys \
  -H "Authorization: Bearer <api-key>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "my-macbook",
    "public_key": "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAI... user@my-macbook"
  }'
```

## 响应示例

```json
{
  "id": "key_xyz789",
  "name": "my-macbook",
  "fingerprint": "SHA256:abc123...",
  "created_at": "2024-01-10T08:00:00Z"
}
```
